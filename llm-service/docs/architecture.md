# Архитектура

Стартовый профиль использует один локальный Ray node и одну RTX 5090 на Ubuntu
24.04. Один процесс `vllm serve` подключается к этому node и поднимает
OpenAI-совместимый API на приватном адресе LLM-хоста.

```text
mail-agent -> 192.168.88.251:8001/v1 -> vLLM driver -> Ray -> GPU 0 (RTX 5090)
```

Используется:

```text
tensor_parallel_size = 1
pipeline_parallel_size = 1
distributed_executor_backend = ray
language_model_only = true
```

`Qwen/Qwen3.5-9B` является мультимодальной моделью, но агент передаёт только
текст. Флаг `--language-model-only` не загружает vision-часть и оставляет больше
памяти для языковой модели. На RTX 5090 модель загружается в GPU, поэтому
`--cpu-offload-gb` установлен в `0`.

Вся операционная логика находится в Makefile. `config.mk` содержит только
несекретные параметры. `VLLM_API_KEY` и `HF_TOKEN` передаются в окружении процесса.
Шаблон worker сохранён для отдельного двухузлового расширения, но не участвует в
развёртывании на одной RTX 5090.
