# Архитектура

## Компоненты

```text
HTTP-запрос
  → FastAPI-роут
  → ProcessingService
  ├─ CapabilitiesRegistry
  ├─ FileProcessor
  ├─ ModelManager
  │   └─ адаптер PaddleOCR / PP-StructureV3
  ├─ InferenceLimiter
  └─ нормализатор результата
  → JSON-ответ
```

| Компонент | Ответственность |
| --- | --- |
| Роуты FastAPI | Принимают multipart-запросы, получают `request_id`, вызывают сервисный слой и документируют API в OpenAPI. |
| `CapabilitiesRegistry` | Единый реестр задач, моделей, языков, значений по умолчанию и совместимых комбинаций. Используется и для `/api/v1/capabilities`, и для валидации запросов. |
| `FileProcessor` | Проверяет размер, MIME-тип, расширение, сигнатуру и целостность изображения/PDF; удаляет временные PDF после запроса. |
| `ModelManager` | Лениво загружает модели, защищает от одновременной повторной загрузки, ведёт LRU-кэш и освобождает вытесненные экземпляры. |
| Адаптеры PaddleOCR | Инкапсулируют официальный Python API `PaddleOCR` и `PPStructureV3`. |
| `InferenceLimiter` | Выполняет блокирующий инференс в потоке, ограничивает общее число параллельных операций и сериализует доступ к одному экземпляру модели. |
| Нормализаторы | Преобразуют внутренние результаты PaddleOCR в стабильные Pydantic-схемы публичного API. |

## Жизненный цикл приложения

При запуске FastAPI создаёт настройки, реестр возможностей, обработчик файлов, менеджер моделей и ограничитель инференса. Тяжёлые модели при этом не загружаются.

Проверка `GET /health/live` не выполняет I/O, не загружает модели и не запускает инференс. `GET /health/ready` проверяет настройки, доступность каталогов модели и временных файлов, а также валидность реестра; она также не загружает модели.

При завершении приложения `ModelManager` освобождает неиспользуемые кэшированные конвейеры.

## Обработка запроса

1. Middleware принимает или генерирует `X-Request-ID`.
2. Реестр проверяет задачу, модель, язык и их совместимость.
3. `FileProcessor` читает файл порциями, не превышая лимит, и валидирует содержимое.
4. `ModelManager` возвращает кэшированный конвейер либо создаёт его один раз для данного ключа.
5. `InferenceLimiter` выполняет инференс вне event loop.
6. Нормализатор формирует публичный ответ без внутренних объектов PaddleOCR.
7. Временный файл PDF удаляется независимо от исхода обработки.

## Ключ кэша и параллелизм

Ключ кэша включает задачу, идентификатор модели, язык, устройство Paddle и параметры конвейера. Для каждого ключа есть отдельная блокировка загрузки: два одновременных запроса не создадут два экземпляра одной модели.

Размер LRU-кэша задаёт `MODEL_CACHE_SIZE`. Вытесняется наименее недавно использованный экземпляр без активных запросов. Активный экземпляр освобождается только после завершения последней аренды.

Ограничение `MAX_CONCURRENT_INFERENCES` действует на весь процесс. Дополнительно у каждого экземпляра модели есть отдельная блокировка инференса, поскольку потокобезопасность PaddleOCR не предполагается.

## Хранение данных и безопасность

- Пользовательские документы не сохраняются постоянно.
- Изображения передаются в PaddleOCR в памяти.
- PDF получает случайное имя во временном каталоге и удаляется в `finally`.
- В логах нет имени файла, содержимого документа, распознанного текста и бинарных данных.
- Клиенту не возвращаются трассировки, локальные пути и необработанные исключения PaddleOCR.
