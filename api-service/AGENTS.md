# Правила для Results API

- Сервис независим от `agent/`: взаимодействие допускается только через HTTP-контракт из `docs/api.md`.
- Не пишите в журналы MIME, тела писем, заголовки, имена вложений, результаты OCR/LLM, ключи API или S3-учётные данные.
- Хранилище PostgreSQL использует partitioned таблицы; новые запросы к `emails` и `email_attachments` обязаны иметь ограничение `received_at`.
- Бинарные объекты передаются потоково через MinIO. Не добавляйте прямые S3 URL в JSON-ответы.
- Перед изменением API обновляйте тесты и русскую документацию. Регулярные тесты не требуют Docker.
